# Customer Service Ticketing — Execution Plan

## Overview

Implement Customer Service Ticketing following the same Clean Architecture + CQRS pattern used across the CRM. A Ticket tracks a customer support request linked optionally to an Account and a Contact. `TicketNumber` is auto-generated server-side on creation (format: `TKT-00001`) — it is not a form field.

---

## Phase 1: Domain Layer

### 1.1 Ticket Priority Enum
**File:** `src/CRM.Domain/Enums/TicketPriority.cs`

```
Low, Medium, High, Critical
```

### 1.2 Ticket Status Enum
**File:** `src/CRM.Domain/Enums/TicketStatus.cs`

```
Open, InProgress, Resolved, Closed
```

### 1.3 Ticket Entity
**File:** `src/CRM.Domain/Entities/Ticket.cs`

Extend `BaseEntity`. Fields:
- `TicketNumber` (string, required — auto-generated, format: `TKT-00001`)
- `Subject` (string, required)
- `AccountId` (Guid, nullable — FK to Account)
- `ContactId` (Guid, nullable — FK to Contact)
- `Priority` (TicketPriority enum, default: Medium)
- `Status` (TicketStatus enum, default: Open)
- `IsDeleted` (bool, default: false)

Navigation properties:
- `Account` (Account, nullable)
- `Contact` (Contact, nullable)

## Phase 2: Application Layer

### 2.1 DTOs
**File:** `src/CRM.Application/Features/Tickets/TicketDtos.cs`

- `TicketSummaryDto` — Id, TicketNumber, Subject, Priority, Status (for list/grid)
- `TicketDetailDto` — all fields including AccountId, AccountName, ContactId, ContactName (for get by id / edit form)

### 2.2 Create Ticket
**Folder:** `src/CRM.Application/Features/Tickets/CreateTicket/`

- `CreateTicketCommand.cs` — `IRequest<Guid>` record (Subject, AccountId, ContactId, Priority, Status)
  - `TicketNumber` is not in the command — generated by the handler via `ITicketRepository.GetNextTicketNumberAsync()`
- `CreateTicketValidator.cs` — FluentValidation: Subject required, Priority required
- `CreateTicketHandler.cs` — calls `GetNextTicketNumberAsync`, maps command → Ticket entity (with generated TicketNumber), calls `ITicketRepository.AddAsync`, returns Id

### 2.3 Update Ticket
**Folder:** `src/CRM.Application/Features/Tickets/UpdateTicket/`

- `UpdateTicketCommand.cs` — `IRequest` record (Id + Subject, AccountId, ContactId, Priority, Status)
  - `TicketNumber` is excluded — it is immutable after creation
- `UpdateTicketValidator.cs` — same rules as create
- `UpdateTicketHandler.cs` — fetches ticket via `GetByIdAsync`, updates fields, calls `UpdateAsync`

### 2.4 Get Ticket By Id
**Folder:** `src/CRM.Application/Features/Tickets/GetTicketById/`

- `GetTicketByIdQuery.cs` — `IRequest<TicketDetailDto>` record (Id)
- `GetTicketByIdHandler.cs` — calls `GetByIdAsync`, throws `NotFoundException` if null, maps to `TicketDetailDto`

### 2.5 Search Tickets
**Folder:** `src/CRM.Application/Features/Tickets/GetTickets/`

- `GetTicketsQuery.cs` — `IRequest<PagedResult<TicketSummaryDto>>` record (Search, Page = 1, PageSize = 10)
- `GetTicketsHandler.cs` — calls `GetPagedAsync`, maps entities to `TicketSummaryDto`

### 2.6 Delete Ticket
**Folder:** `src/CRM.Application/Features/Tickets/DeleteTicket/`

- `DeleteTicketCommand.cs` — `IRequest` record (Id)
- `DeleteTicketHandler.cs` — fetches ticket, calls `SoftDeleteAsync`

### 2.7 Repository Interface
**File:** `src/CRM.Application/Interfaces/ITicketRepository.cs`

Methods:
- `GetByIdAsync(Guid id)`
- `GetPagedAsync(string? search, int page, int pageSize)`
- `GetNextTicketNumberAsync()` — returns next formatted ticket number (e.g., `TKT-00042`)
- `AddAsync(Ticket ticket)`
- `UpdateAsync(Ticket ticket)`
- `SoftDeleteAsync(Guid id)`

## Phase 3: Infrastructure Layer

### 3.1 EF Core Configuration
**File:** `src/CRM.Infrastructure/Persistence/Configurations/TicketConfiguration.cs`

- Table name: `Tickets`
- Required fields: TicketNumber, Subject
- `TicketNumber`: unique index, max length 20
- Default value for Priority: Medium
- Default value for Status: Open
- `AccountId`: nullable FK to `Accounts` table (set null on delete)
- `ContactId`: nullable FK to `Contacts` table (set null on delete)

### 3.2 DbContext Update
**File:** `src/CRM.Infrastructure/Persistence/AppDbContext.cs`

Add: `DbSet<Ticket> Tickets`

### 3.3 Ticket Repository
**File:** `src/CRM.Infrastructure/Repositories/TicketRepository.cs`

Implement `ITicketRepository`:
- `GetByIdAsync` — include `Account` and `Contact` navigation properties
- `GetPagedAsync` — filter by search on TicketNumber, Subject; exclude soft-deleted; order by CreatedAt desc (newest first)
- `GetNextTicketNumberAsync` — count all tickets (including soft-deleted) and return `TKT-{(count + 1):D5}` to guarantee uniqueness
- `SoftDeleteAsync` — set `IsDeleted = true`

### 3.4 DI Registration
**File:** `src/CRM.API/Program.cs`

Add: `builder.Services.AddScoped<ITicketRepository, TicketRepository>();`

### 3.5 Database Migration
Run after all infrastructure changes:
```
dotnet ef migrations add AddTicketTable --project src/CRM.Infrastructure --startup-project src/CRM.API
dotnet ef database update --project src/CRM.Infrastructure --startup-project src/CRM.API
```

## Phase 5: Frontend

### 5.1 API Client
**File:** `web/src/api/tickets.ts`

Types: `TicketSummary`, `TicketDetail`, `CreateTicketPayload`, `UpdateTicketPayload`, `TicketsPagedResult`

Functions:
- `getTickets(params: { search?, page?, pageSize? })`
- `getTicketById(id: string)`
- `createTicket(payload: CreateTicketPayload)`
- `updateTicket(id: string, payload: UpdateTicketPayload)`
- `deleteTicket(id: string)`

### 5.2 Ticket List Page
**File:** `web/src/pages/Tickets/TicketListPage.tsx`

Features:
- Search input with debounce
- Paginated MUI Table with columns: Ticket Number, Subject, Priority, Status
- "Add Ticket" button → navigates to `/tickets/new`
- Edit icon → navigates to `/tickets/:id/edit`
- Delete icon → opens `DeleteConfirmDialog`, calls `deleteTicket`
- React Query (`useQuery`) for data fetching with cache invalidation on mutation

### 5.3 Ticket Form Page
**File:** `web/src/pages/Tickets/TicketFormPage.tsx`

Shared create/edit form:
- Fields:
  - Subject (text input, required)
  - Account (MUI Select populated from `GET /api/accounts`, optional)
  - Contact (MUI Select populated from `GET /api/contacts`, optional)
  - Priority (MUI Select from `TicketPriority` enum values, required)
  - Status (MUI Select from `TicketStatus` enum values)
- React Hook Form for form state
- Ticket Number is display-only on the edit form (read from `TicketDetailDto`, not editable)
- On load (edit): fetch ticket by id, populate form
- On submit: call `createTicket` or `updateTicket` via `useMutation`, redirect to list on success
- Client-side validation: Subject required, Priority required

### 5.4 Routing
**File:** `web/src/App.tsx` (or router config)

Add routes:
```
/tickets              → TicketListPage
/tickets/new          → TicketFormPage (create mode)
/tickets/:id/edit     → TicketFormPage (edit mode)
```

### 5.5 Navigation
**File:** `web/src/components/Layout/` (sidebar/nav component)

Add "Tickets" nav item linking to `/tickets`.
